AI Coding 作业要点 - API 通知系统

目标与背景
- 设计并实现一个“API 通知系统”，对内提供统一、标准化的 HTTP API。
- 业务系统只需传入目标地址与请求内容；系统负责稳定、可靠地投递到外部系统。
- 允许使用 AI 辅助分析/设计/编码，重点在工程决策与取舍。

提交物要求
- README 或设计文档：问题理解、整体架构与核心设计、关键工程决策与取舍。
- 代码实现：可为最小可行实现（MVP），不要求完整生产级。
- AI 使用说明：AI 帮助点、未采纳建议、自主关键决策及原因。
- GitHub 仓库命名：rc_{your_nickname}。

需求与场景
- 多业务系统触发关键事件时需通知外部供应商 HTTP(S) API。
- 供应商 API 差异：地址、Header、Body 格式不同。
- 业务系统不关心外部 API 返回值，但需确保通知尽可能可靠送达。

设计说明需明确回答
- 系统边界：解决哪些问题、不解决哪些问题及原因。
- 可靠性与失败处理：投递语义（至少一次等）；外部系统失败/不可用的处理策略。
- 取舍与演进：哪些 AI 建议过度设计并不采纳；未来流量/复杂度增长的演进方向与依据。
- 中间件/基础设施（若使用）：选择原因与替代方案。

评估重点
- 问题拆解与系统边界界定。
- 复杂度管理与取舍。
- 对 AI 输出的判断、修正与取舍能力。
- 设计是否体现真实工程经验。
